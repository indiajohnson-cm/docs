Opciones e indicadores¶

Argumentos¶

-h, --help

Mostrar mensaje de ayuda y salir.

--images <path>, -i <path>

Ruta a las imágenes de entrada.

--project-path <path>

Ruta a la carpeta del proyecto.

--resize-to <integer>

Cambia el tamaño de las imágenes por el lado más grande para OpenSFM. Configure en -1 para deshabilitar.

Predeterminado: 2048

--end-with <string>, -e <string>

Opciones: dataset | split | merge | opensfm | mve | odm_filterpoints | odm_meshing | mvs_texturing | odm_georeferencing | odm_dem | odm_orthophoto

--pc-ept

Genera una Nube de Puntos Entwine. Útil por sí solo, este indicador se agrega automáticamente cuando se ejecuta en WebODM y es útil para crear la estructura de directorio correcta cuando se ejecuta en NodeODM o en la línea de comandos para importar a WebODM.

--rerun <string>, -r <string>

Opciones: dataset | split | merge | opensfm | mve | odm_filterpoints | odm_meshing | mvs_texturing | odm_georeferencing | odm_dem | odm_orthophoto

--rerun-all

Forzar la repetición de todas las tareas.

--rerun-from <string>

Opciones: dataset | split | merge | opensfm | mve | odm_filterpoints | odm_meshing | mvs_texturing | odm_georeferencing | odm_dem | odm_orthophoto

--proj <PROJ4 string>

Proyección utilizada para transformar el modelo a coordenadas geográficas.

--min-num-features <integer>

Número mínimo de características para extraer por imagen. Mas características llevan a mejores resultados pero una ejecución mas lenta.

Predeterminado: 8000

--feature-quality

Seleccionar la calidad de extracción de rasgos. Una calidad mas alta genera mejores rasgos, pero requiere mas memoria y toma mas tiempo.

Opciones: ultra | high | medium | low | lowest

Predeterminado: high

--matcher-neighbors <integer>

Número de imágenes cercanas a la comparación previa basado en los datos exif de GPS. Establezca en 0 para omitir la comparación previa. El parámetro Neighbors trabaja junto con el parámetro Distance, establezca ambos en 0 para no utilizar la comparación previa. OpenSFM usa ambos parámetros al mismo tiempo, Bundler usa solo uno que tiene valor, prefiriendo el parámetro Neighbors.

Predeterminado: 8

--matcher-distance <integer>

Umbral de distancia en metros, para encontrar imágenes pre-coincidentes basado en los datos exif de GPS. Establezca ambos matcher-neighbors y este en 0 para saltar la comparación previa.

Predeterminado: 0

--use-fixed-camera-params

Desactive la optimización de parámetros de cámara durante la agrupación. Desactivado de manera predeterminada, a menos que se utilice el parámetro --camera.

--camera-lens <string>

Configure un tipo de proyección de cámara. Establecer manualmente un valor puede ayudar a mejorar la distorsión geométrica. De forma predeterminada, la aplicación intenta determinar el tipo de lente a partir de los metadatos de las imágenes.

Opciones: auto | perspective | brown | fisheye | spherical

Predeterminado: auto

--radiometric-calibration <string>

Configure la calibración radiométrica a realizar en las imágenes. Al procesar imágenes multiespectrales, debe configurar esta opción para obtener valores de reflectancia (de lo contrario, obtendrá valores numéricos digitales (DN)).

[camera] aplica nivel de negro, viñeteo, ganancia de gradiente de fila / compensación de exposición (si se encuentran las etiquetas EXIF apropiadas).

[camera+sun] es experimental, aplica todas las correcciones de la [camera] y además compensa la radiación espectral registrada a través de un sensor de luz descendente (DLS) teniendo en cuenta el ángulo del sol.

Opciones: none | camera | camera+sun

Predeterminado: none

--max-concurrency <positive integer>

El número máximo de procesos a utilizar en varios procesos. El requisito de memoria máxima es ~1 GB por hilo y una resolución de imagen de 2 megapíxeles.

Predeterminado: number of cores

--depthmap-resolution <positive float>

Controla la densidad de la nube de puntos estableciendo la resolución de las imágenes del mapa de profundidad. Los valores más altos tardan más en calcularse y requieren más memoria, pero producen nubes de puntos más densas.

Predeterminado: 640

--opensfm-depthmap-min-consistent-views <integer: 2 ≤ x ≤ 9>

Número mínimo de vistas que deben reconstruir un punto para que sea válido. Utilice valores más bajos si sus imágenes tienen menos traslape. Los valores más bajos dan como resultado nubes de puntos más densas pero con más ruido. Solo se aplica si se usa OpenSfM para coincidencias densas.

Predeterminado: 3

--opensfm-depthmap-method <string>

Algoritmo de cálculo de mapas de profundidad sin procesar. PATCH_MATCH y PATCH_MATCH_SAMPLE son más rápidos, pero pueden perder algunos puntos válidos. BRUTE_FORCE lleva más tiempo pero produce reconstrucciones más densas.

Opciones: PATCH_MATCH | PATCH_MATCH_SAMPLE | BRUTE_FORCE

Predeterminado: PATCH_MATCH

--opensfm-depthmap-min-patch-sd <positive float>

Cuando se usa PATCH_MATCH o PATCH_MATCH_SAMPLE, se controla el umbral de la desviación estándar para incluir parches. Los parches con una desviación estándar mas baja se ignoran.

Predeterminado: 1

--use-hybrid-bundle-adjustment

Ejecute un ajuste de paquete local para cada imagen agregada a la reconstrucción y un ajuste global cada 100 imágenes. Acelera la reconstrucción de conjuntos de datos muy grandes.

--mve-confidence <float: 0 ≤ x ≤ 1>

Descarta puntos que tienen menos de un cierto nivel de confianza. Esto afecta solamente a las reconstrucciones densas hechas con MVE. Los valores mas altos descartan mas puntos.

Predeterminado: 0.6

--use-3dmesh

Utilice una malla 3D completa para calcular la ortofoto en vez de utilizar una mala 2.5D. Esta opción es un poco mas rápida y produce resultados similares en áreas planas.

--skip-3dmodel

Omitir la generación de un modelo 3D completo. Esto puede ahorrar tiempo si solamente necesita resultados 2D como ortofotos o DEMs.

--use-opensfm-dense

Utilice openSFM para calcular la nube de puntos densa, de forma alternativa.

--ignore-gsd

Ignore la distancia de muestreo del suelo (GSD). GSD limita la resolución máxima de las imágenes de salida y cambia el tamaño de las imágenes cuando es necesario, lo que resulta en un procesamiento más rápido y un menor uso de memoria. Dado que el GSD es una estimación, a veces ignorarlo puede resultar en una calidad de imagen de salida ligeramente mejor.

--mesh-size <positive integer>

El conteo máximo de vértices de la malla de salida.

Predeterminado: 100000

--mesh-octree-depth <positive integer>

La profundidad del Oct-tree utilizada en la reconstrucción de la malla, aumente para obtener más vértices, los valores recomendados son 8-12.

Predeterminado: 9

--mesh-samples <float ≥ 1.0>

Número de puntos por nodo octree. Para un ruido relativamente bajo, se pueden usar valores en el rango de [1-5]. Para conjuntos de datos con mas ruido, se pueden usar valores mas grandes en el rango de [15-20].

Predeterminado: 1.0

--mesh-point-weight <positive float>

Este valor de punto flotante especifica la importancia que se le da a la interpolación de las muestras puntuales en la formulación de la ecuación de Poisson filtrada. Los resultados de la reconstrucción de Poisson original (no filtrada) se pueden obtener estableciendo este valor en 0.

Predeterminado: 4

--fast-orthophoto

** Experimental.** Omite la reconstrucción densa y la generación del modelo 3D. Genera una ortofoto directamente a partir de la reconstrucción escasa. Si solo necesita una ortofoto y no necesita un modelo 3D completo, active esta opción.

--crop <positive float>

Recorte automáticamente los resultados de la imagen creando un búfer suave alrededor de los límites del conjunto de datos, reducido en N metros. Utilice 0 para deshabilitar el recorte.

Predeterminado: 3

--pc-classify

Clasifica las nubes de puntos de salida usando un filtro morfológico simple. Puede controlar el comportamiento de esta opción modificando los parametros --dem-*.

Predeterminado: False

--pc-csv

Exportar la nube de puntos georeferenciada en formato CSV.

Predeterminado: False

--pc-las

Exportar la nube de puntos georeferenciada en formato LAS.

Predeterminado: False

--pc-filter <positive float>

Filtra la nube de puntos eliminando puntos que se desvían mas de N desviaciones estándar del promedio local. Configure en 0 para deshabilitar el filtrado.

Predeterminado: 2.5

--smrf-scalar <positive float>

Parámetro escalar de elevación del Filtro Morfológico Simple.

Predeterminado: 1.25

--smrf-slope <positive float>

Parámetro de pendiente del filtro morfológico simple (altura sobre longitud).

Predeterminado: 0.15

--smrf-threshold <positive float>

Parámetro de umbral de elevación del Filtro Morfológico Simple (metros).

Predeterminado: 0.5

--smrf-window <positive float>

Parámetro de radio de la ventana de Filtro Morfológico Simple (metros).

Predeterminado: 18.0

--texturing-data-term <string>

Opciones: area | gmi

Predeterminado: gmi

--texturing-nadir-weight <integer: 0 ≤ x ≤ 32>

Afecta solo a las ortofotos. Los valores más altos dan como resultado esquinas más nítidas, pero pueden afectar la distribución del color y el desenfoque. Utilice valores más bajos para áreas planas y valores más altos para áreas urbanas. El valor predeterminado funciona bien para la mayoría de los escenarios.

Predeterminado: 16

--texturing-outlier-removal-type <string>

Tipo de método de eliminación de valores atípicos fotométricos.

Opciones: none | gauss_damping | gauss_clamping

Predeterminado: gauss_clamping

--texturing-skip-visibility-test

Omitir prueba de visibilidad geométrica.

Predeterminado: False

--texturing-skip-global-seam-leveling

Omitir la nivelación global de costura. Útil para datos IR.

Predeterminado: False

--texturing-skip-local-seam-leveling

Omitir el mezclado de costuras locales.

Predeterminado: False

--texturing-skip-hole-filling

Omitir el llenado de huecos en la malla.

Predeterminado: False

--texturing-keep-unseen-faces

Mantener en la malla las caras que no se ven en ninguna cámara.

Predeterminado: False

--texturing-tone-mapping <string>

Active el mapeo de tonos gamma o ninguno para no mapear tonos.

Opciones: gamma | none

Predeterminado: none

--gcp <path string>

Ruta al archivo que contiene los puntos de control sobre el terreno utilizados para la georreferenciación. El archivo debe tener el siguiente formato de línea: easting northing height pixelrow pixelcol imagename

Predeterminado: None.

--use-exif

Use esta etiqueta si tiene un gcp_list.txt pero desea usar las geoetiquetas exif en su lugar.

--dtm

Utilice esta etiqueta para construir un DTM (modelo digital de terreno, solo suelo) utilizando un filtro morfológico simple. Verifique los parámetros --dem* y --smrf* para un ajuste más fino.

--dsm

Utilice esta etiqueta para construir un DSM (modelo de superficie digital, suelo + objetos) utilizando un filtro morfológico progresivo. Verifique los parámetros --dem* para un ajuste más fino.

--dem-gapfill-steps <positive integer>

Número de pasos utilizados para rellenar áreas con huecos. Configure en 0 para deshabilitar el llenado de espacios. Comenzando con un radio igual a la resolución de salida, se generan N DEM diferentes con un radio progresivamente mayor utilizando el algoritmo de distancia inversa ponderada (IDW) y se fusionan. Luego, los espacios restantes se fusionan utilizando la interpolación del vecino más cercano.

Predeterminado: 3

--dem-resolution <float>

Resolución del DSM/DTM en cm 7 pixel.

Predeterminado: 5

--dem-decimation <positive integer>

Diezma los puntos antes de generar el DEM. 1 no diezma (calidad total). 100 diezma ~ 99% de los puntos. Útil para acelerar la generación.

Predeterminado: 1

--dem-euclidean-map

Calcula un mapa ráster euclidiano para cada DEM. El mapa informa la distancia desde cada celda al valor NODATA más cercano (antes de que se llene cualquier agujero). Esto puede resultar útil para aislar las áreas que se han llenado.

Predeterminado: False

--orthophoto-resolution <float: x ﹥0.0>

Resolución de la ortofoto en cm / pixel.

Predeterminado: 5

--orthophoto-no-tiled

Configure este parámetro si quiere un geoTIFF por franjas.

Predeterminado: False

--orthophoto-compression <string>

Establezca la compresión que se utilizará. Tenga en cuenta que esto podría afectar gdal_translate si no sabe lo que está haciendo.

Opciones: JPEG | LZW | PACKBITS | DEFLATE | LZMA | NONE

Predeterminado: DEFLATE

--orthophoto-bigtiff <string>

Controle si la ortofoto creada es un BigTIFF o un TIFF clásico. BigTIFF es una variante para archivos de más de 4GiB de datos. Consulte las especificaciones de GDAL: https://www.gdal.org/frmt_gtiff.html para obtener más información.

Opciones: YES | NO | IF_NEEDED | IF_SAFER

Predeterminado: IF_SAFER

--orthophoto-cutline

Genera un polígono alrededor del área de recorte que corta la ortofoto alrededor de los bordes de las rasgos. Este polígono puede ser útil para unir mosaicos sin costura con varias ortofotos superpuestas.

Predeterminado: False

--build-overviews

Crea descripciones generales de ortofoto con gdaladdo.

--verbose, -v

Imprime mensajes adicionales en la consola.

Predeterminado: False

--time

Genera un archivo de referencia con información del tiempo de ejecución.

Predeterminado: False

--version

Muestra el número de versión del software y sale.

--split <positive integer>

Número medio de imágenes por submodelo. Al dividir un conjunto de datos grande en submodelos más pequeños, las imágenes se agrupan en grupos. Este valor regula la cantidad de imágenes que debe tener cada grupo en promedio.

--split-overlap <positive integer>

Radio de superposición entre submodelos. Después de agrupar imágenes en grupos, las imágenes que están más cerca de este radio de un grupo se agregan al grupo. Esto se hace para asegurar que los submodelos vecinos se superpongan.

--optimize-disk-space

Elimine archivos intermedios pesados (como ortofotos originales, dtm, dsm) para optimizar el uso del espacio en disco, manteniendo las versiones comprimidas. Esto afecta la capacidad de reiniciar la canalización desde una etapa intermedia, pero permite que los conjuntos de datos se procesen en máquinas que no tienen suficiente espacio en disco disponible. Además, en este modo, los «informes» no se escriben en la carpeta de salida “opensfm”.

Predeterminado: False

--sm-cluster <string>

URL a una instancia de ClusterODM para distribuir un flujo de trabajo de split-merge en varios nodos en paralelo.

Predeterminado: None

--merge <string>

Elije qué combinar en el paso de combinación de un conjunto de datos dividido. De forma predeterminada, se fusionan todas los archivos salidas disponibles.

Predeterminado: all
